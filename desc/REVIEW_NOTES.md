# 기획 주석 보드 — 검수 메모

이 보드를 만들면서 눈에 띈, **제품/문서 결정이 필요한 항목**입니다. 보드 내용 자체(오탈자·설명 오류)는 바로 고쳤고, 여기 남긴 것은 코드/디자인 의도를 확인해야 할 사안입니다.

## 1. 행사 담당자 — 매장 현황 화면의 상태 필터 탭 (desc/boards/17-eventManagerStores.html, 핀 3)
전체/영업중/일시중지/마감 4개 탭인데, 모바일 프레임(402px)에서는 마지막 "마감" 탭이 화면 폭에 딱 맞지 않아 가로로 살짝 스크롤해야 보일 수 있습니다. 다른 화면의 3개짜리 세그먼트 탭(대기/접수/완료 등)은 문제없이 들어갑니다.
**확인 필요:** 의도된 가로 스크롤(모바일 탭 UI에서 흔한 패턴)인지, 탭 4개를 더 압축해야 하는지.

## 2. "완료 처리 (0회)" 라벨의 0회 표기 (customers-received, 핀 3)
되돌리기 없이 처음 완료 처리하는 정상적인 경우에도 버튼에 항상 "(0회)"라는 숫자가 보입니다. '고객 호출 (N회)'와 표기 방식을 맞추려는 의도는 이해되지만, 0회일 때도 굳이 숫자를 보여줄지는 UX 카피 관점에서 재확인해볼 만합니다.
**확인 필요:** 0회일 때는 숫자를 생략하고 1회 이상부터 "(N회)"를 붙이는 게 나을지.

## 3. 행사 담당자 홈 — 주의가 필요한 매장 문구의 비현실적인 경과 시간 (eventManagerHome, 핀 5)
캡처 시점 기준 "대기 주문이 889분째 접수되지 않고 있어요" 같이 매우 큰 숫자가 노출되는 경우가 있었습니다. 이는 목업 시드 데이터의 상대 시각과 실제 캡처 시점 사이 시간차 때문에 생긴 데모 데이터 특성이며, 실제 서비스 데이터에서는 발생하지 않을 상황입니다.
**확인 필요 없음** — 참고용으로만 남깁니다.
